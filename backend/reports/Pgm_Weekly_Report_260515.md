## 章節 5：重大決策與戰略討論

**決策一：OpenMAM 2.10 合版策略確立（5/13）**

OpenMAM 3.0 功能 merge 進 2.x 分支，版本號命名為 2.10，預計 6 月中完成合併、6 月底整合驗證。此決策讓 TVBS 得以在既有 2.x 架構上獲得 3.0 bug fix 與功能，不打亂既有部署。

**決策二：AI Server Phase 1 / Phase 2 分工確立（5/13）**

Phase 1（6 月）優先完成人臉識別整合，軟體先行，DB migration 問題同步處理不作為 blocker。Phase 2 開始前，Steve 必須先完成全功能 AI Server 架構設計，涵蓋 STT engine、NLP engine、字幕工具等。

**決策三：STT 差異化策略——繞過準確率天花板，走編輯 UX（5/13）**

全球 STT 準確率上限約 90% 出頭，不再只拼準確率，改在使用者體驗上做出差異：快速斷點校正、timecode 對齊，可包裝第三方 STT 疊加自研編輯工具。校對工時比準確率更容易量化，也更直接影響記者的實際作業效率。

**決策四：小栗方 Lite 短期以 POC 方式獨立運行（5/13）**

學習吧 / 加分吧整合暫不列入，因目前對核心功能影響有限。短期先以獨立 POC 方式驗證核心功能，整合議題留待後續再議。

**決策五：涉及台灣官員的 Demo 場合暫不使用 AI 語音功能（5/13）**

5/13 嘉義 demo 中發現 AI 模型帶有大陸口音及用語（例如「坐公交車」），在台灣政治敏感場合幾乎是 demo 當場失分。決定：凡涉台官員、政治敏感場合的 demo，暫不使用可能輸出大陸用語的 AI 語音功能。此原則需全員落實。

**決策六：新竹縣客語 ASR 案——必須有示意圖才進入提案（5/13）**

縣府不接受單純報價，至少需要 demo 或網頁示意圖，才能在局內討論並向縣府報告。本週確認先做示意圖，再推進報價與提案。
