## 章節 5：重大決策與戰略討論

**決策一：客語辨識廢標底線確立**
黎博明確表示，若客語辨識（客委會+新竹縣）內部成本無法壓低至可負擔範圍，將主動放棄合約（廢標）。此為財務紀律原則的明確宣示，優先保護公司 margin。

**決策二：學習吧串接策略轉向 API Spec**
原討論的「整合進內部」或「遷移」方案均因工程量過大而排除。改以開放 API Spec 定義，讓外部系統在學習吧「外部」完成串接，大幅降低 RD 依賴與成本。

**決策三：OlaMedia 以 Solo Product 角度先行規劃**
暫不等待 OpenMAM 整合架構定案，先以 2B 獨立產品角度規劃 OlaMedia。即便 DGX Q2 Ready，OlaMedia 整合進 OpenMAM 供用戶使用最樂觀估計仍需至 Q4。

**決策四：AI NLE 定位為輕量級 OpenEdit Plus**
不取代 Avid/Media Composer 等專業 NLE，定位為小編/記者的 AI 輔助快速剪輯工具，以 Chat 介面為 UX 演進目標，Q3 目標 Alpha/Beta。

**決策五：OpenMAM 整合路線（加法 vs 減法）5/13 討論後定案**
加法（2.0+AI）適合既有用戶升級；減法（3.0 精簡）適合 AI-first 版本。需 JH 加入才能具體推進，單方無法決定。

**決策六：威栗雲台灣更新 10 man-days 待核准**
黎博原本認為只是「改幾個字」，Tonny 評估後確認需 10 man-days（差距約 10 倍）。此差異需 Michael 負責溝通說明，確認後方可動工。

**決策七：5/3 硬體 Demo 改為 API 後台展示**
大陸端硬體無法在台灣平台完成 register，放棄實體硬體展示，改以 VS Code 環境 API 後台呈現；同時請大陸端提供連線操作示範影片。

**決策八：Shirene 串接需求採兩階段處理**
先正式回覆可執行串接（承諾）、請對方提供窗口，後續再安排實際工程執行，避免立即阻塞研發資源。

**決策九：創造栗內容本地化不動用大陸 RD**
課程積木簡→繁轉換待大陸端平台完成後，台灣端自行接手，遵循公司規範，不佔用大陸 RD 資源。
